A quick overview of testing Rest APIs using RestAssured Java

What is an API

API stands for Application Programming Interface . Consider an application we have hotels booking , whenever a person visits a new city we want our website to be visible in flights booking website so that the customers can book the hotel rooms from this hotels booking site

In order to provide the rooms availability in hotel , we need to communicate with the flights booking website instead of providing complete code base to the flights website we will provide an api that only retrieves information about the availability of rooms in the hotels website

Thus we are providing an api to communicate between multiple sites , similarly we can use apis to communicate between the frontend and backend of application

So it becomes very much necessary to be able to provide righteous information , thus we need to test these APIs

REST stands for Representational State Transfer

HTTP methods

HTTP methods are used to communicate with REST APIs

The most commonly used http methods are GET POST PUT DELETE

GET method is used to retrieve information from the server

POST method is used to send data to the server

PUT method is used to update the existing data

DELETE method is used to delete the data

Lets understand the terms in URI

EndPoint URL: baseURI/resource/(Query/Path)Parameters

baseURI the domain of the website like - google.com is considered as baseURI

resource represent API/collection which can be accessed from server

google.com/maps here maps is resource

google.com/search here search is resource

google.com/images here images is resource

Path parameters

Path parameters are variable parts of a URL path which are used to point a specific resource within a collection

google.com/images/1121212 here 1121212 is path parameter pointing a specific resource here image

Query parameters

Query parameters is used to sort/filter the resources , identified with ?

https://shop.com/orders?sort_by=2/11/2005 here ?sort_by=2/11/2005 is query parameter

Headers/Cookies

Headers represents the meta data associated with API request and response we are sending additional details to API to process our request
